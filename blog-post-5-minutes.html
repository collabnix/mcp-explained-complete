How I Built My First MCP Server in 5 Minutes

A beginner's journey into Model Context Protocol development

? May 24, 2025 ? ?? 5 min read ? ? Beginner Friendly
TL;DR: I thought building an MCP server would take hours or days. Turns out, it took me exactly 5 minutes to have a working server that could integrate with Claude and provide custom context. Here's exactly how I did it, step by step.

? What I Thought vs. Reality

When I first heard about Model Context Protocol (MCP), I imagined complex server configurations, databases, and hours of debugging. I was completely wrong.

MCP servers are surprisingly simple: they're just programs that can provide context, tools, or resources to AI models like Claude. Think of them as smart plugins that make AI more powerful by giving it access to specific information or capabilities.

? The 5-Minute Challenge

I decided to time myself building my first MCP server. My goal: create a simple server that could provide information about my favorite programming languages. Here's exactly what happened:

1 Setup (1 minute) 1:00

First, I created a new directory and installed the MCP SDK: 

mkdir my-first-mcp-server
cd my-first-mcp-server
npm init -y
npm install @modelcontextprotocol/sdk
2 Basic Server Structure (2 minutes) 2:00

I created server.js with the basic MCP server structure: 

import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';

// My simple data store
const programmingLanguages = {
  python: {
    description: "A versatile, easy-to-learn language perfect for beginners",
    useCase: "Web development, data science, AI, automation",
    difficulty: "Beginner-friendly"
  },
  javascript: {
    description: "The language of the web, runs everywhere",
    useCase: "Web development, mobile apps, server-side development",
    difficulty: "Beginner to intermediate"
  },
  rust: {
    description: "Fast, safe systems programming language",
    useCase: "Systems programming, web backends, blockchain",
    difficulty: "Advanced"
  }
};

// Create the server
const server = new Server(
  {
    name: "programming-languages-server",
    version: "1.0.0"
  },
  {
    capabilities: {
      resources: {}
    }
  }
);
3 Add Resource Handler (1 minute) 3:00

Added the logic to handle requests for programming language information: 

// Handle resource requests
server.setRequestHandler('resources/list', async () => {
  return {
    resources: Object.keys(programmingLanguages).map(lang => ({
      uri: `language://${lang}`,
      name: `${lang.charAt(0).toUpperCase() + lang.slice(1)} Programming Language`,
      description: programmingLanguages[lang].description,
      mimeType: "text/plain"
    }))
  };
});

server.setRequestHandler('resources/read', async (request) => {
  const uri = request.params.uri;
  const language = uri.replace('language://', '');
  
  if (programmingLanguages[language]) {
    const langInfo = programmingLanguages[language];
    const content = `
Language: ${language.toUpperCase()}
Description: ${langInfo.description}
Primary Use Cases: ${langInfo.useCase}
Difficulty Level: ${langInfo.difficulty}
    `.trim();
    
    return {
      contents: [{
        uri,
        mimeType: "text/plain",
        text: content
      }]
    };
  }
  
  throw new Error(`Language ${language} not found`);
});
4 Start the Server (30 seconds) 3:30

Added the startup code at the end of the file: 

// Start the server
async function main() {
  const transport = new StdioServerTransport();
  await server.connect(transport);
  console.error("Programming Languages MCP Server running!");
}

main().catch(console.error);
5 Test It Works (1.5 minutes) 5:00

Added the server to my Claude config and tested it: 

# In Claude's MCP settings, I added:
{
  "mcpServers": {
    "programming-languages": {
      "command": "node",
      "args": ["path/to/my-first-mcp-server/server.js"]
    }
  }
}

? SUCCESS! Total Time: 5 Minutes

My MCP server was running and Claude could now access my programming language knowledge base!

? What I Learned

Building my first MCP server taught me several important things:

  • Simplicity wins: MCP servers are just programs that respond to specific requests
  • JSON is your friend: Everything communicates via simple JSON messages
  • Resources are powerful: Even simple data becomes valuable context for AI
  • Testing is instant: Changes show up immediately in Claude
  • Scalability is built-in: Easy to add more data or functionality
